## 目标
将 `scout-lab` 从“项目拼装态”推进到“可治理、可调度、可回测、可运营”的情报 agents 工作树，并为后续多线程/多 agent 协作提供统一任务协议。

## 当前工作树
- 代码目录: `/Users/sourcefire/1data/scout-lab`
- 主仓库: `https://github.com/biexiaofeeng-boop/scout.cn-agents`
- 当前主线分工:
  - `scout-hub`: 事件接入、健康状态、告警与轻量数据面
  - `scout-media-agents`: 选题治理、seed/expansion/query/schedule 控制面
  - `scout-wchat-agents`: 微信链路与任务生产
  - `scout-vendor/mediacrawler`: 平台 crawler 执行面
  - `scout-deploy`: 启动、容器编排、环境检查

## 推荐分支策略

- 子任务分支建议:
  - `codex/scout-media-governance-docs`

  - `codex/scout-media-governance-cli`

## 执行规则
1. 每个线程或 agent 只认领一个任务卡。
2. 先改代码或文档，再回写任务卡状态：`TODO -> DOING -> CHECK -> DONE`。
3. 每次提交必须补齐：改动文件列表、验证命令、验证结论。
4. `scout-vendor/mediacrawler` 视为 vendor 运行面，不在治理任务中做破坏性重构。
5. 任何会影响运行态的变更，必须在 `02-Checks-验收清单.md` 与当期任务包中补回归命令。

## 合并规则
1. 文档、schema、CLI 三类改动可分提交，但必须最终回到同一主线。
2. 与运行环境相关的变更，必须附带回滚办法。
3. 未通过 smoke / check 的改动不得推送主线。

1. `scout-media-agents` 不再做“随意爬”，而是做治理控制面。
