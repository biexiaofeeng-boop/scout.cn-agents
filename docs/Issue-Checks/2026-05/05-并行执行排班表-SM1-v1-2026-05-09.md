# 并行执行排班表：SM1

## 目标
明确后续如需多 agent 并行开发，如何拆分 `scout-media-agents` 的治理控制面任务，避免重复改同一文件。

## Agent 分工建议
### Agent-A 文档与治理设计
- 负责范围:
  - `docs/Issue-Checks/**`
  - 设计稿、任务包、检查清单
- 不改动:
  - `scout-media-agents/src/**`

### Agent-B Domain Schema / Registry
- 负责范围:
  - `scout-media-agents/src/types.ts`
  - `scout-media-agents/src/*registry*.ts`
  - `scout-media-agents/src/*state*.ts`
- 不改动:
  - CLI 输出层

### Agent-C CLI / UX
- 负责范围:
  - `scout-media-agents/src/cli.ts`
  - 命令帮助、参数解析、输出格式
- 依赖:
  - 等 Agent-B 定义 domain 接口后接入

### Agent-D 验证与回测
- 负责范围:
  - `npm run check`
  - CLI smoke
  - 回填 `Checks` 文档与任务卡状态
- 不改动:
  - 业务实现

## 合作规则
1. 每个 agent 只改自己责任文件。
2. 先接口后实现，避免 CLI 和 schema 双向猜测。
3. 所有命令最终收口到主线程统一提交。
